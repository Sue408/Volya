## 1. 设计理念

### 1.1 核心定位

Volya 是一款 **AI 驱动的小说创作桌面应用**。与通用 AI 聊天工具不同，它的核心场景是 **长时间的沉浸式创作**。因此设计上需要平衡：

| 维度 | 偏向 |
|------|------|
| 专注度 | 鼓励沉浸，减少视觉干扰 |
| 温暖感 | 有陪伴感，不冰冷 |
| 功能性 | 信息可及，不藏匿 |
| 个性 | 有独特的视觉记忆点 |

### 1.2 五条设计原则

```
① 简化至上
   从复杂开始，有意识删减到最简单的有效方案。
   每个多余的线条、颜色、文字都在消耗用户的注意力。

② 功能分层
   用字号、颜色、间距来创造层次，而非靠花哨装饰。
   最重要的信息最突出，次要的信息退后。

③ 细节偏执
   每个像素都是有意识的选择。
   对齐、间距、色彩一致性——这些决定品质感。

④ 语言统一
   每个元素都视觉上传达其功能。
   用户看到卡片就知道可交互，看到按钮就知道可点击。

⑤ 技术隐形
   用户专注于创作和与 Doro 的对话，而非界面本身。
   UI 应当是透明的——好到用户注意不到它的存在。
```

### 1.3 情感基调

> "像是坐在洒满阳光的书房里，桌边有一杯热茶，Doro 就坐在对面陪你一起构思故事。"

- **温暖的** — 鼠尾绿 + 暖杏橙，自然的色彩让人放松
- **安静的** — 大量留白，不喧哗
- **有活力的** — Doro 的小表情和微交互带来生命力
- **专业的** — 克制的排版和间距，不幼稚

### 2.2 字体体系 (Typography)

#### 字体选择

| 用途 | 字体 | 回退 | 权重 |
|------|------|------|------|
| **标题/展示** | `'Newsreader'` | `Georgia, serif` | 400~700 |

| **代码/数据** | `'JetBrains Mono'` | `'SF Mono', monospace` | 400~500 |

> `Newsreader` 是一款优雅的衬线字体，有文学气质，适合标题和展示场景。

## 3. 设计 Token 体系

### 3.1 语义 Token 映射

所有组件应使用**语义 Token**（如 `--bg-primary`），而非直接引用色阶 Token（如 `--sage-50`）。
这样在深色模式切换时只需更新 Token 映射，组件无需改动。

```css
/* 背景层级 */
--bg-primary:       var(--sage-50);     /* 最底层页面背景 */
--bg-secondary:     #ffffff;            /* 表面/卡片背景 */
--bg-tertiary:      var(--sage-100);    /* hover、次要背景 */
--bg-card:          #ffffff;            /* 卡片背景 */
--bg-input:         #ffffff;            /* 输入框背景 */
--bg-chat:          #ffffff;            /* 聊天区域背景 */
--bg-hover:         var(--sage-100);    /* 悬浮态背景 */
--bg-titlebar:      var(--sage-100);    /* 标题栏背景 */

/* 文字层级 */
--text-primary:     var(--neutral-900); /* 主要文字 */
--text-secondary:   var(--neutral-600); /* 次要文字 */
--text-tertiary:    var(--neutral-400); /* 辅助文字、占位符 */
--text-link:        var(--sage-600);    /* 链接文字 */
--text-on-color:    #ffffff;            /* 深色背景上的文字 */

/* 边框层级 */
--border-color:     var(--sage-200);    /* 标准边框 */
--border-light:     var(--sage-100);    /* 浅边框 */
--divider-color:    var(--sage-100);    /* 分割线 */

/* 强调色 */
--accent-primary:   var(--sage-500);    /* 主强调色 */
--accent-secondary: var(--apricot-400); /* 次强调色 🍊 */
--accent-hover:     var(--sage-600);    /* 强调色 hover */
```

### 4.5 全局组件：TitleBar

**统一的标题栏组件，在所有页面复用。**

```
┌────────────────────────────────────────────────────┐
│  [左侧区]           [中间区]          [右侧区]       │
│  ← 返回 / 🍊 图标  页面标识           状态 + 窗口控制 │
└────────────────────────────────────────────────────┘
```

| 区域 | 内容 | 技术 |
|------|------|------|
| 左侧 | 返回按钮（非首页）或应用图标 | `-webkit-app-region: no-drag` |
| 中间 | 页面路径标识 | `-webkit-app-region: drag`（可拖拽） |
| 右侧 | 连接状态 + 权限等级 + 最小化/最大化/关闭 | `-webkit-app-region: no-drag` |

**窗口控制按钮：**

```
[ ─ ] 最小化    → 调用 window.minimize()
[ □ ] 最大化/还原 → 调用 window.toggleMaximize()
[ ✕ ] 关闭      → 调用 window.close()
```

## 6. 交互与动效

### 6.1 微交互动效清单

| 场景 | 动效 | 时长 | 方式 |
|------|------|------|------|
| 消息出现 | 淡入 + 向上 8px 位移 | 300ms | CSS `@keyframes` |
| 按钮 hover | 轻微加深 + 缩放 1.02 | 150ms | `transition` |
| 按钮点击 | 缩放 0.97 | 100ms | `transition` |
| 卡片 hover | 上浮 2px + 阴影加深 | 200ms | `transition` |
| 侧栏展开/收起 | 宽度渐变 | 250ms | CSS `transition` |
| 页面切换 | 淡入 | 200ms | Vue `<Transition>` |
| 审批按钮 | 点击后"盖章"轻弹效果 | 300ms | `@keyframes` |
| Doro 思考 | 小点脉冲呼吸 | 1.5s loop | `@keyframes pulse` |
| 输入框聚焦 | 边框色变化 + 光晕 | 200ms | `transition` |
| 新消息指示 | 底部出现"新消息"小标签 | 自动 | 仅当未在最底部时 |

### 6.2 动画命名规范

```css
/* 出现 */
@keyframes fadeIn  { ... }     /* 淡入 */
@keyframes slideUp { ... }     /* 上滑 */
@keyframes scaleIn { ... }     /* 缩放出现 */

/* 持续 */
@keyframes pulse   { ... }     /* 呼吸脉冲 */
@keyframes shimmer { ... }     /* 加载闪烁 */
@keyframes bounce  { ... }     /* 弹跳（极克制使用） */
```

### 6.3 动效原则

- **克制** — 动效是为了传达状态，不是炫技
- **快速** — 所有动效不超过 300ms，不让人等待
- **自然** — 使用 `ease` / `ease-out`，不突兀
- **可降级** — 用户 `prefers-reduced-motion` 时禁用动画
